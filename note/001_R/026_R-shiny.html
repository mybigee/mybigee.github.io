1.2 用户界面（UI）

  • ui定义APP的输入层和输出层，server函数则计算输出以及任何需要计算的输出。

fluidPage()

  • UI的最外层函数，该函数创建一个由行和列构成的流动的页面布局。
  • 行确保其中的元素出现在同一行。这些行中的列定义每个元素应占据多少水平空间。
  • 流体页面实时缩放其组件以填充所有可用的浏览器宽度，APP开发人员无需担心为各个APP组件定义相对宽度。

sidebarLayout()

  • 创建一个侧边栏的输入层。
  • Shiny 包含许多用于布置APP组件的选项，示例APP中使用的默认布局是带有侧边栏的布局，可以在sidebarLayout()定义。

  • 一个简单的布局，具有用于输入的狭窄侧边栏和用于输出的较宽的主区域。
  • 在底层，Shiny 实现了 Bootstrap 2 中提供的布局功能。

3.2 使用反应式

  • 为什么要使用反应式：

    • 在前面的练习中，在将子集数据帧定义为反应式表达式后，我们能够在服务器的多个位置重用它。
    • 一般来说，反应式表达式可以帮助您避免复制粘贴代码，它们还有助于将大型复杂的计算分解为较小的部分。

  • 函数和反应式：

    • 每次调用函数时，R 都会对其求值。
    • 然而反应式表达式是惰性的，它们只有在输入改变时才会被执行。这意味着即使您在应用程序中多次调用响应式表达式，它也只会在其输入更改时重新执行。

  • 查看反应日志：

    • 启动一个新的 R 会话，然后运行options(shiny.reactlog = TRUE)
    • 启动应用程序
    • 然后在应用程序中按Ctrl + F3（或在 Mac 上：Cmd + F3）

  • 练习：3-2a Find missing reactives、3-2b Find inconsistencies in what the app is reporting

3.4 停止触发延迟

  • 在本节中，介绍如何停止、触发和延迟 Shiny 操作。

  • 隔离反应：假设现在有一个小部件，用户可以输入绘图的标题。

    只有当input$x或input$y发生变化的时候，plot和标题都会更新，而只输入标题的时候，plot不会更新。

x
output$scatterplot <- renderPlot({
  ggplot(data = movies_subset(), aes_string(x = input$x, y = input$y)) +
    geom_point() + labs(title = isolate({ input$plot_title}))
})
  • 触发反应：使用observeEvent()（事件）

  • 第一个参数为反应式，第二个参数是当第一个参数无效时要调用的表达式。
  • 假设您的应用程序允许根据样本大小的数字输入对数据进行随机采样。同时用户在按下请求的操作按钮 ( actionButton())时下载生成的随机样本

  • 在UI中定义了一个按钮，同时observeEvent()通过ID绑定了该按钮，之后只要点击该按钮，就会执行第二个参数的{}中的代码。

  • 延迟反应：使用eventReactive()，参数与observeEvent()类似。

    假设您的目标是改变用户在您的应用程序中获取随机样本的方式 - 您只希望他们在按下“获取新样本”的操作按钮时获取新样本，而不是在定义大小的数字输入等其他内容时获取新样本。

  • ignoreNull参数会告诉R当事件结果为NULL的时候做与不做，例如，当应用程序首次启动且用户尚未与应用程序交互时，应用程序应该做什么？如果设置为FALSE，应用程序将首先执行操作或计算，然后用户可以重新启动它。

  • observeEvent()和eventReactive()

    • observeEvent()用于执行响应事件的操作
    • eventReactive()用于创建仅响应事件而更新的计算值

  • observeEvent()/eventReactive()与observe()/reactive()

    • observe()并且reactive()功能会在他们访问的任何内容上自动触发
    • observeEvent()并且eventReactive()需要明确告知函数触发它们的原因

  • isolate()与事件处理函数

    • isolate()用于停止反应
    • observeEvent()用于执行响应事件的操作
    • eventReactive()用于创建仅响应事件而更新的计算值

  • 练习：3-4a Stop with isolate()、3-4b Delay with eventReactive()、3-4c Trigger with observeEvent()

conditionalPanel()

  • 创建一个根据输入或输出值而可见的面板。

xxxxxxxxxx
library(shiny)


# Define UI with conditionalPanel
ui <- fluidPage(
  titlePanel("Random number generator"),
  sidebarLayout(
    sidebarPanel(
      selectInput(
        "digit_type", "Number of digits:",
        c("any", "selected")
      ),
      conditionalPanel(
        condition = "input.digit_type == 'selected'",
        sliderInput("digit_count", "How many digits?",
                    min = 1, max = 10, value = 4
        )
      ),
      actionButton("go", "Generate new random number"),
      width = 5
    ),
    mainPanel(
      br(),
      "Your random number is",
      h4(textOutput("random_number")),
      width = 7
    )
  )
)


# Define server logic that generates a random number based on user input
server <- function(input, output, session) {
  output$random_number <- renderText({
    input$go
    raw <- runif(1)
    digits <- if (input$digit_type == "any") {
      sample(1:10, size = 1)
    } else {
      input$digit_count
    }
    round(raw * 10^digits)
  })
}


shinyApp(ui = ui, server = server)

选项卡和反应

  • 如果数据集的计算成本很高，那么如果每个选项卡都需要进行计算，则用户界面的渲染速度可能会很慢。
  • 相反，我们在反应式表达式中计算一次数据，然后让所有输出选项卡共享结果。

navlistPanel()
