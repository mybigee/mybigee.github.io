SummarizedExperiment对象

  • SummarizedExperiment是一种利用R语言对象存储基因表达数据的常见方式，它可以对不同来源的实验数据进行整合，将基因的基本信息、表达数据以及其他辅助信息统一保存在一个对象中，方便后续的数据处理和分析。该对象通常包含以下三个部分：
  1. assay：存储基因表达矩阵，可以是RNA-seq或者microarray技术得到的原始计数数据或标准化后的表达值。
  2. rowData/colData：分别存储基因或样本的元数据信息，如基因名称、基因注释、样本ID、组别等。
  3. metadata：存储与数据集相关的元数据信息，如数据来源、描述、版本、作者、更新时间等。
  • 使用SummarizedExperiment对象可以实现简单而高效的数据存储和处理，也适合不同的R包进行基因表达数据分析。同时，该对象也能够与其他常见的R语言包进行无缝集成，如limma、edgeR、DESeq2等。

SingleCellExperiment对象

  • SingleCellExperiment是一种用于存储和操作单细胞转录组数据的R语言对象，它是基于SummarizedExperiment和DelayedArray类开发的。与SummarizedExperiment不同，SingleCellExperiment专门用于存储单细胞RNA测序（scRNA-seq）数据，并提供了许多针对单细胞数据的方法和函数，可以方便地进行数据预处理、维度缩减、差异表达分析等常见数据处理操作。
  • SingleCellExperiment对象中通常包含以下信息：
  1. 表达矩阵：储存基因表达计数或RPKM等值，每行表示一个基因，每列表示一个单细胞样本。
  2. meta-data信息：关于基因和细胞的注释信息，如基因名称、注释、单细胞ID、细胞类型等。
  3. QC信息：质控和过滤后的数据信息，如过滤掉的细胞或基因的ID列表、UMI分布图等。
  4. 基因集注释：储存外部基因注释信息，如基因集名字、相应基因集内的基因名等。
  • 使用SingleCellExperiment，可以完成各种基于单细胞数据的分析，包括绘制tSNE/UMAP图、发现关键基因、鉴定细胞亚类等等。

ExpressionSet对象

  • ExpressionSet是Bioconductor R软件包中的一个重要数据类，用于存储基因表达数据以及与之相关的元数据和注释信息。ExpressionSet的设计初衷主要是为了处理芯片数据，但现在也被广泛用于RNA-seq等其他类型的高通量数据。
  • 该对象提供了一种方便的方式来组织和操作高维基因表达数据，以及它们所对应的样本和基因信息。ExpressionSet对象包含以下三个主要成分：
  1. exprs: 基因表达矩阵，行表示基因，列表示样本。
  2. phenoData: 包含每个样本的元数据，如样本来源、处理条件等。
  3. featureData: 包含基因的注释信息，如基因名、描述、位置等。
  • 使用ExpressionSet，我们可以方便地进行数据预处理、可视化和统计分析。例如，对exprs矩阵进行标准化、过滤、差异表达分析，或者根据phenoData信息进行tSNE/PCA聚类分析等。常用的函数包括 subset, pData, fData等。

DESeqDataSet对象

  • DESeqDataSet对象是DESeq2软件包中的一个类，用于存储差异表达分析的输入数据、中间计算结果和输出结果。它是RangedSummarizedExperiment的子类，代表了处理后的样本数据和与之相关的注释信息。
  • 在DESeq2中，差异表达分析的核心是针对每个基因定量转录本或者外显子序列的读数计数。这些计数必须存储为DESeqDataSet对象中的矩阵，其中第一组数据是一个称为“counts”的矩阵，它仅包含非负整数值。这个矩阵表示了每个样本在每个基因或转录本水平上的读数计数。
  • 除了“counts”矩阵之外，DESeqDataSet对象还包括其他关键信息，如实验设计公式、样品元数据以及基因或转录本级别的注释信息等。

cpm()和cpm

  • cpm()为edgeR包里的一个函数，用于计算基因在不同样本中的counts per million（CPM）。它可以将每个样本中的每个基因的计数数据转换为CPM值，并进行库大小归一化。
  • cpm指的是counts per million，它是基因表达计数数据的一种常用的归一化和转换方法，用于比较在不同样本中基因表达的相对水平。它通过将每个样本中的每个基因的计数值除以总读数，并乘以1,000,000得到。相当于将计数数据转换为每百万个reads中该基因的计数值。
  • 这种归一化方法可以解决两个样本之间由于库大小不同而引起的表达计数不可比的问题。同时，它也可以避免在过度表达的基因上产生偏差。因此，在进行差异基因分析时，通常会使用cpm值来代替原始计数数据进行进一步的数据分析。

library(edgeR)

head(cpm(expSet),3)
# 进行标准化，因为可能存在0，因此+1

exprs()

  • exprs()是R语言中用于提取 ExpressionSet对象的基因表达矩阵的函数之一。该函数通常结合其他R软件包使用，并且可用于从分子生物学实验数据中提取差异表达基因信息。
  • 在Bioconductor R软件包中， exprs()函数通常用于从 ExpressionSet对象中提取基因表达值，并生成一个基因表达矩阵，其中每行对应一个基因，每列对应一个样本（或实验条件）。

model.matrix()

# 设计矩阵

makeContrasts()

  • makeContrasts()函数是R语言中的一个函数，通常用于在做差异分析（如RNA-Seq等）时构建线性模型，并将需要比较的不同条件之间的差异编码成适合于线性模型的对照矩阵。该函数主要用于基因表达谱数据分析，它可以将需要比较的各个条件之间的差异转化为对应的对比系数和对照矩阵。
xxxxxxxxxx
makeContrasts(contrast_name, levels)

lmFit()

  • lmFit()函数是R语言中的一个函数，通常用于基因表达谱数据分析中，用于构建线性模型并对其进行拟合。该函数主要是为了在差异表达分析等任务中使用，通过将所需比较的对照矩阵应用于基因表达谱数据中的基因表达量值，从而获得差异表达的统计显著性。
xxxxxxxxxx
lmFit(y, design)
xxxxxxxxxx
fit1 <- lmFit(experSet,design)

contrasts.fit()

  • contrasts.fit() 是一个统计数据分析中用于比较和检查数据组之间差异的函数。在使用线性模型时，可以使用 contrasts.fit() 函数对因子变量进行编码和处理，以便更好地理解和分析变量之间的关系。
xxxxxxxxxx
# 构建线性模型并对其进行拟合
fit1 <- lmFit(experSet,design)
# 统计数据分析中用于比较和检查数据组之间差异的函数
fit2 <- contrasts.fit(fit1,contrast.matrix)

eBayes()

  • limma 包提供的函数，用于基于基因表达数据进行贝叶斯假设检验（Bayesian hypothesis testing）。该函数针对 limma 中的线性模型对象（linear model object） 进行贝叶斯假设检验。
  • 通过运行 eBayes() 函数，可以获得每个基因的概率值（posterior probability），这些概率值反映了基因的显著性和差异性。在分析基因表达数据时，eBayes() 可以帮助我们进一步验证哪些基因受到影响，并确定它们是否显著。
xxxxxxxxxx
# 构建线性模型并对其进行拟合
fit1 <- lmFit(experSet,design)
# 统计数据分析中用于比较和检查数据组之间差异的函数
fit2 <- contrasts.fit(fit1,contrast.matrix)
# 进行贝叶斯检验
fit3 <- eBayes(fit2)

topTable()

  • limma 包提供的函数，用于从线性模型（linear model）对象中提取差异或显著性最高的基因或蛋白质等分子列表。常见的应用场景是，在进行差异表达分析后，筛选出表达水平发生显著改变的基因列表。
  • 参数：fit 是一个已经进行过 t 检验或者 t-变换（t-transformation）的线性模型对象。coef 参数表示要提取哪个系数（coefficient），通常为 1 表示提取待比较组与对照组之间的差异值。n 参数则表示想要提取的差异或显著性最高的分子数量，默认为 Inf 表示获取所有分子。
xxxxxxxxxx
# 构建线性模型并对其进行拟合
fit1 <- lmFit(experSet,design)
# 统计数据分析中用于比较和检查数据组之间差异的函数
fit2 <- contrasts.fit(fit1,contrast.matrix)
# 进行贝叶斯检验
fit3 <- eBayes(fit2)
# 提取出全部的差异表达基因
tempOutput <- topTable(fit3,coef = 1,n = Inf)
nrDEG <- na.omit(tempOutput)
