数据库

  • GO（Gene Ontology）：一个用于描述基因和蛋白质功能的标准化分类体系，旨在统一描述所有生命形式中的基因和蛋白质的功能和相互关系。它是一种结构化的词汇，按照三个级别对生命过程中的基本结构、分子功能和生物学过程进行分类。http://geneontology.org/
  • KEGG（Kyoto Encyclopedia of Genes and Genomes）：一个综合性的数据库，它整合了生物化学、基因组学、代谢学和系统生物学等多个领域的数据和信息，并提供了一系列工具和资源，以帮助研究人员更好地理解生物和其调控机制。https://www.genome.jp/kegg/
  • reactome：一个用于生物学领域的免费开源数据库，提供了生物分子和化学反应的详细信息。Reactome 数据库主要关注代谢、信号传导和基因调控三个方面，并提供了包括人类、小鼠、大鼠、果蝇、斑马鱼、蛋白质等多种不同物种的数据。https://reactome.org/
  • MSigDB（Molecular Signatures Database）：一个基因集数据库，这些基因集包括来自文献报道的已知的代谢途径、信号通路、基因本体和其他生物过程的基因集，以及从基因组大规模数据中发现的基因集，例如转录因子结合位点和差异表达的基因集等。https://www.gsea-msigdb.org/gsea/msigdb

# enrichGO()
ego_ALL <- enrichGO(gene = gene1$ENTREZID,
                    universe = names(geneList), # 背景基因集
                    OrgDb = org.Hs.eg.db, # 指定物种和注释信息
                    ont = "ALL", # GO的三个类别都富集分析
                    pAdjustMethod = "BH", # 使用BH的方法校正p值
                    pvalueCutoff = 1, # p值的过滤阈值，为1表示不过滤
                    qvalueCutoff = 1,
                    readable = T) # 将gene的ID转为symbol，便于阅读，否则转换后的结果在geneID仍然为ENTREZID
View(summary(ego_ALL))

结果解读

  • ONTOLOGY：富集到的GO类型，CC、BP、MF；
  • ID：GO数据库中的ID；
  • Description：GO功能；
  • GeneRatio：富集到该GO条目的差异基因（gene1，自己的基因集）数与差异基因总数的比值；
  • BgRation：富集到GO该条目背景基因（geneList）数与背景基因总数的比值；
  • pvalue：富集的显著性水平；
  • p.adjust：调整后的P值；
  • qvalue：FDR值，错误发现率，越小越好；
  • geneID：富集到该Go条目的基因；
  • Count：富集到该Go条目的基因数。

GO富集分析

分析

  • enrichGO()：函数使用超几何分布来计算基因集是否受到富集，更适用于使用经典的富集统计方法。
  • gseGO()：函数则使用图形化显示富集情况的基因集富集评分（Gene Set Enrichment Score，GSES）来计算富集，更适用于使用基于排名的富集分析方法。
xxxxxxxxxx
# GO分析
# 使用超几何的方法
ego <- enrichGO(gene = gene,
                universe = names(geneList), # 背景基因集
                OrgDb = org.Hs.eg.db,
                ont = "CC", # 细胞组分
                pAdjustMethod = "BH",
                pvalueCutoff = 0.01, # 保留的p值的阈值
                qvalueCutoff = 0.05,
                readable = TRUE) # 将ID转为symbol便于阅读
View(summary(ego))

结果解读

  • ID：GO数据库中的ID；
  • Description：GO功能；
  • setSixe：被分析的基因集（输入的基因列表）在该 GO 条目中的基因数目；
  • enrichmentScore：基因集富集评分（Gene Set Enrichment Score，GSES），表示该基因集在该 GO 条目中的富集程度；
  • NES：标准化富集评分（Normalized Enrichment Score，NES）；
  • pvalue：显著性水平；
  • p.adjust：经过多重检验校正后的 p 值；
  • qvalue：加权多重检验校正后的 p 值；
  • rank：根据 GSES 对富集结果进行排序的排名；
  • leading_edge：富集结果的前导边界，表示富集分析中得到的基因集的核心成员。tags表示核心基因在该基因集基因总数的占比，而list表示核心基因占所有基因总数的比例，signal利用这两个指标计算得到。
  • core_enrichment：核心富集基因集。
