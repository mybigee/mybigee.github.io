reads比对到参考基因组

介绍

  • 使用小鼠1号染色体来说明使用R如何进行read比对和计数。

  • 许多比对软件可以使用：

    • bowtie
    • topHat
    • STAR
    • Rsubread

  • Rsubread是唯一可以在R允许的比对软件，通常来说，比对软件需要消耗大量的计算资源，同时一般都运行在Linux。

  • 对于普通的笔记本而言，许多比对过程要求更大的计算量。因此比对工作经常运行在Linux服务器中。

reads文件

  • 小鼠乳腺数据（fastq文件）：一共12个样本，https://figshare.com/s/f5d63d8c265a05618137，使用校园网下载可能遇到403 Forbidden此时切换为流量下载。
  • 该数据集的GEO：https://www.ncbi.nlm.nih.gov/geo/query/acc.cgi?acc=GSE60450
  • 原始的reads来自SRA，文件的格式是.sra，NCBI提供的sra toolkit可以使用fastq-dump命令将.sra的数据转为.fastq格式的数据。对应的SRA链接为https://ncbi.nlm.nih.gov/sra/?term=SRP045534

下载基因组文件

  • 想要获取完整的基因组fastq文件，可以访问：

    • UCSC genome browser：http://hgdownload.soe.ucsc.edu/downloads.html
    • NCBI：http://www.ncbi.nlm.nih.gov/genome
    • ENSEMBL：http://asia.ensembl.org/info/data/ftp/index.html

创建索引

  • reads被存储在压缩的fastq文件在，在进行差异分析之前，要先进行比对（alignment）到小鼠基因组同时计数（count）到注释基因。该过程可以使用Rsubread包完成。
  • 比对的第一步先创建索引（index）。这里只创建1号染色体的索引，创建全基因组的索引需要耗费较多的时间，因此在此处有提供索引文件。
  • 以下命令mm10的1号染色体（chr1）信息存储在Mus_musculus.GRCm39.dna.chromosome.1.fa文件中。
xxxxxxxxxx
# 创建1号染色体的索引
buildindex(basename = "chr1_mm10",reference = "C:/Users/SYWS/Desktop/Mus_musculus.GRCm39.dna.chromosome.1.fa")
  • 上面的命令会在工作目录中产生索引文件，索引文件的前缀为chr1_mm10，使用dir()参考当前工作目录下的文件。

reads比对到参考基因组

  • 使用align()函数，将reads比对到参考基因组。
  • 如果我们有PE（paired end）数据，需要指定第二个参数readfile2。
  • 我们此处的小鼠数据为100pb的SE（single end）数据。
  • 我们可以指定输出文件的文件名，默认的是在文件名的后面添加.subread.BAM。
xxxxxxxxxx
# 比对到参考基因组，指定工作目录下 chr1_mm10 开头的索引文件
align(index="chr1_mm10",readfile1 = fastq.files)
  • 上面命令会逐个比对12个样本。由于我们只使用每个样本1000个读数的子集，因此每个样本的比对应该只需要几分钟左右的时间。要运行此数据集中的完整样本，每个样本需要几个小时。
  • BAM 文件保存在reads文件所在的同一个目录中。
  • 此时每个样本生成的文件有3个：bam文件、cvf文件、比对结果的汇总文件

计数

  • 计数（counting）
  • 到目前为止，我们已经找出每个reads来自基因组的哪一个位置。
  • 小鼠reads计数对应到基因可以通过featureCounts()函数进行计数，该函数包含的注释有：小鼠（mm39、mm9、mm10）和人类（hg38、hg19），基于的注释是NCBI RefSeq。
  • 下面的代码使用在mm10基因组的NCBI refseq注释中定义的外显子间隔。将配对（map）到基因外显子的reads相加以获得每个基因的计数，并注意跨越外显子和外显子边界的reads。
  • 输入是一些列的BAM文件，输出是一个counts矩阵的对象，每行是一个基因（Entrez ID），每列是一个样本。
xxxxxxxxxx
# 小鼠基因注释的版本指定mm39，多比对
# 使用annot.ext参数可以自己提供注释文件（GTF）
fc <- featureCounts(bam.files,annot.inbuilt = "mm39",isPairedEnd = F)

与教程的差异

  • 在教程中align()的默认是进行唯一的reads比对（unique = TURE），而现在该函数默认是进行多比对（unique = FALSE）
  • 在教程在featureCounts()函数在进行计数的时候，也是默认进行唯一比对（countMultiMappingReads = FALSE），而现在该函数是默认进行多比对（countMultiMappingReads = TRUE），同时默认是SE（isPairedEnd = FALSE）。

提示

  • 几乎所有的测序仪提供的测序数据都是fastq格式的文件。
  • 对于来自GEO/SRA的公开可以的测序数据，一般的默认格式是SRA，需要使用SRA Toolkit的fastq-dump将该格式转为fastq。https://www.ncbi.nlm.nih.gov/books/NBK158900（可能已过时）。
  • 对于illumina1.8以上的版本Phred数值为+33，而旧的版本为+64。
  • Rsubread包含小鼠和人类的内置基因注释。对于其他物种，用户需要读取GTF格式的数据框来定义基因和外显子。用户还可以指定 SAF 格式的自定义注释文件。
