Introduction to scRNA-seq integration（scRNA-seq整合）

简介

  • 联合分析两个或多个单细胞数据集存在一定的挑战，特别是在跨不同的数据集中识别细胞类群可能存在问题。
  • Seurat v4包含一系列的方法去匹配跨数据集共享细胞类群。
  • 方法首先识别处于匹配生物学状态（锚点）的跨数据集的配对细胞，既可以用于校正不同数据集之间的技术差异（批次效应），也可以跨实验条件的scRNA-seq比较分析。
  • 以对处于静止或干扰素刺激状态的人免疫细胞（PBMC）进行比较分析。

集成的目标

  • 创建一个整合的数据集（assay）为下游的分析
  • 识别两个数据集中存在的细胞类型
  • 获得在刺激和控制组中都保守的细胞类型marker
  • 比较数据集找到细胞类型对刺激的特异性反应

使用SCTransform进行数据集标准化的集成

  • 使用一种基于正则化负二项式回归的改进的scRNA-seq标准化的方法。

  • 与原来的流程大致相同，不同在于：

    • 标准化数据集单独使用SCTransform()，不需要在集成之前使用NormalizeData()。
    • 通常使用3000个或以上的feature来进行sctransform的下游分析。
    • 在找出锚点之前使用PrepSCTIntegration()。
    • 使用FindIntegrationAnchors()和IntegrateData()的时候将方法设置为SCT。
    • 不需要运行ScaleData()。
xxxxxxxxxx
# 划分数据集
ifnb.list <- SplitObject(ifnb, split.by = "stim")
# 数据标准化
ifnb.list <- lapply(X = ifnb.list, FUN = SCTransform)
features <- SelectIntegrationFeatures(object.list = ifnb.list, nfeatures = 3000)
ifnb.list <- PrepSCTIntegration(object.list = ifnb.list, anchor.features = features)


# 找出锚点，方法使用SCT
immune.anchors <- FindIntegrationAnchors(object.list = ifnb.list, 
        normalization.method = "SCT", anchor.features = features)
# 创建集成的数据集
immune.combined.sct <- IntegrateData(anchorset = immune.anchors, 
                                     normalization.method = "SCT")
# 降维
immune.combined.sct <- RunPCA(immune.combined.sct, verbose = FALSE)
immune.combined.sct <- RunUMAP(immune.combined.sct, reduction = "pca", dims = 1:30)
# 可视化
p1 <- DimPlot(immune.combined.sct, reduction = "umap", group.by = "stim")
p2 <- DimPlot(immune.combined.sct, reduction = "umap", group.by = "seurat_annotations", 
              label = TRUE, repel = TRUE)
