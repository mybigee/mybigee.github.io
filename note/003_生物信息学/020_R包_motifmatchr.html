R包_motifmatchr简介快速开始输入选项背景核苷酸频率对数基数和伪计数P值

简介

  • motifmatchr包被设计为分析许多序列（sequence）或模体（motif），以查找哪些序列包含哪些基序。
  • 它在内部使用使用MOODS C++ library，进行模体匹配。
  • 该包的主要方法是matchMotifs()，它接受motif PWM/PFM 和基因组范围或序列，并返回哪些范围/序列与哪些motif匹配或匹配的位置。
  • 与 Bioconductor 中可用的替代motif匹配函数（Biostrings的matchPWM()或TFBSTools的searchSeq()），motifmatchr专为确定许多不同序列/范围是否包含许多不同基序的用例而设计。
  • 例如，在分析 ChIP-seq 或 ATAC-seq 数据时，人们可能希望找到 JASPAR 数据库等motif集合中的哪些motif出现在哪些peak中。
xxxxxxxxxx
if(!require("motifmatchr")) BiocManager::install("motifmatchr")
library(motifmatchr)

输入

  • 该方法有两个强制参数：

    • 位置权重矩阵（PWM）或位置频率矩阵（PFM），存储在 TFBSTools 包中的 PWMatrix、PFMatrix、PWMatrixList 或 PFMatrixList 对象中。
    • 一组基因组范围（GenomicRanges 或 RangedSummarizedExperiment 对象）或一组序列（DNAStringSet、DNAString 或简单字符向量）。

  • 如果第二个参数是一组基因组范围，则还需要基因组序列。默认情况下使用BSgenome.Hsapiens.UCSC.hg19。

  • 如果使用另一个基因组构建，则应将适合您物种的 BSgenome 对象或适合您物种的 DNAStringSet 或 FaFile 对象传递给参数genome。

选项

背景核苷酸频率

  • 在确定motif匹配时，使用背景核苷酸频率。
  • 默认情况下，背景频率是输入序列的回归树频率 - 要使用替代频率，提供bg参数到match_pwms。
  • 如果输入序列相当短（如我们的插图示例！），则使用其他输入频率可能是有意义的。这里我们展示如何使用
  • 偶数频率：
xxxxxxxxxx
# 偶数频率
motif_ix <- matchMotifs(example_motifs, peaks, genome = "hg19", bg = "even")
  • 使用基因组中的频率：
x
# 使用基因组中的频率
motif_ix <- matchMotifs(example_motifs, peaks, genome = "hg19", bg = "genome")
  • 指定碱基频率：
xxxxxxxxxx
# 指定碱基频率
motif_ix <- matchMotifs(example_motifs, peaks, genome = "hg19",
                        bg = c("A" = 0.2,"C" = 0.3, "G" = 0.3, "T" = 0.2))
  • 如果不指定碱基的名字，那么默认顺序为ACGT。
  • PWMatrix 对象具有可以使用该bg函数访问的关联背景频率。如果提供的背景频率match_pwms与输入 PWM 中的频率不匹配，则调整 PWM 以反映提供的背景频率。计算的分数基于此调整后的 PWM，而不是直接输入 PWM。为了确保使用精确的 PWM 输入计算分数，只需确保传递到 matchMotifs 的背景频率与用于输入 PWM 的软管匹配并存储在 PWMatrix 对象的 bg 槽中。

对数基数和伪计数

  • motifmatchr期望输入 PWM 使用自然对数或 log 2。如果输入是 PFM，则使用 TFBSTools toPWM 来生成具有默认 伪计数 0.8 和以 2 为底的对数的 PWM。为了更好地控制伪计数，只需在调用 matchMotifs 之前使用 toPWM 函数转换 PFM。

P值

  • 默认 p 值截止值为 0.00005。在此 p 值截止值中未对多重比较进行任何调整。该 p 值阈值用于确定分数阈值。

  • matchMotifs 可以返回三种可能的输出，具体取决于out参数：

    • 默认，out = matches：布尔矩阵，指示哪些范围/序列包含哪些motif，存储在motifMatches SummarizedExperiment 对象的assays槽中。motifMatches方法可用于提取布尔矩阵。如果subject参数是 GenomicRanges 或 RangedSummarizedExperiment 对象，或者ranges提供了参数，则返回 RangedSummarizedExeriment 而不是 SummarizedExperiment。
    • out = scores：与 (1) 相同，加上两个额外的assay， 一个矩阵，其中包含每个范围/序列内的高motif得分（仅在存在匹配时才报告分数）和一个包含motif匹配数量的矩阵。motifScores和motifCounts方法可以用于访问这些组件。
    • out = positions：一个 GenomicRangesList，其中包含输入范围/序列内所有匹配的范围。请注意，如果subject参数是字符向量、DNAStringSet 或 DNAString，并且ranges未提供与序列表示的范围相对应的参数，则将返回 IRangesList 对象列表，而不是序列中的相对位置。
