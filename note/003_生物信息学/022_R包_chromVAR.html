简介

  • chromVAR是一个用户分析稀疏染色质可及性的 R 包。chromVAR 将 ATAC-seq 或 DNAse-seq 实验中的对齐片段（过滤重复片段和低质量片段）以及如motif位置等基因组注释作为输入。
  • chromVAR 计算每个注释和每个细胞或样本的偏差，并根据所有细胞或样本的平均值计算可及性与预期可及性的“偏差”。
  • 此教程涵盖了 chromVAR 与标准输入的基本用法。更详细的文档documentation website。更多的描述和应用publication。

读取输入

  • chromVAR将落在开放染色质peak中的片段计数表作为输入。有许多软件包可以根据表观基因组数据创建计数表；chromVAR 还提供了一种专为单细胞 ATAC-seq 计数而定制的方法。

peak

  • 对于使用 chromVAR，建议使用固定宽度、不重叠的峰。该方法对于峰宽度的精确选择相当稳健，但建议宽度为 250 至 500 bp。
  • 如果分析单细胞数据，使用来自批量 ATAC 或 DNAse-seq 数据的峰值是有意义的，这些数据可以来自相同群体或相似群体（或者可能来自公共资源，如Roadmap Epigenomics项目）。
  • 如果组合来自不同群体的多个peak文件，建议将peak组合在一起。filterPeaks()函数（在该小插图中进一步演示）会将peak减少到非重叠集，基于哪个重叠peak在所有数据中具有更强的信号。
  • getPeaks()将peak作为 GenomicRanges 对象读取。我们将通过读取一个小的样本bed文件来展示它的用途。我们将使用sort_peaks参数来表明我们想要对peak进行排序。

过滤输入

  • 如果使用单细胞数据，建议过滤掉读数不足或峰读数比例较低的样品，因为这些可能代表空孔或死细胞。
  • min_in_peaks和min_depth两个参数被使用在过滤中，如果未提供（如上所述），则这些截止值是根据数据中位数估计的。min_in_peaks设置为peak中片段比例的中位数的0.5，min_depth 设置为最大值 500 或中值库大小的 10%。
  • filterSamples()需要将plot = FALSE，不然会绘制一幅图。
xxxxxxxxxx
# 获得要保留的样本索引
counts_filtered <- filterSamples(example_counts, min_depth = 1500, 
                                 min_in_peaks = 0.15, shiny = FALSE)
counts_filtered <- filterSamples(example_counts, min_depth = 1500, 
                                 min_in_peaks = 0.15, shiny = TRUE)

  • 如果shiny参数设置为TRUE（默认值），将会弹出一个shiny小工具，它允许您使用过滤参数并查看哪些单元格通过过滤器或未通过过滤器。

  • motifmatchr包的matchMotifs()可以查找哪一些peak包含哪一些motif，默认情况下返回一个SummarizedExperiment对象，包含一个表明匹配或不匹配的稀疏矩阵。函数要求输入一个基因组序列，可以使用BSgenome、FaFile或DNAStringSet对象提供。
xxxxxxxxxx
library(motifmatchr)
motif_ix <- matchMotifs(motifs, counts_filtered, genome = BSgenome.Hsapiens.UCSC.hg19)
  • 一种选项是 p.cutoff，用于确定motif识别的严格程度。默认值为 0.00005，这往往会为人类motif提供合理数量的motif匹配。
  • 该函数不仅返回motif匹配，还可以返回附加矩阵（存储在assay），其中包含每个peak的motif匹配数和每个peak的最大基序得分。如需此附加信息，请使用out = scores。要返回motif匹配的实际位置，请使用out = positions。out = matches或out = scores的输出可以传递给computeDeviations()函数。
  • 如果不想使用已知的motif，而是想使用确定长度的kmers，可以使用matchKmers()函数，更多细节Annotations section of the documentation website。
xxxxxxxxxx
kmer_ix <- matchKmers(6, counts_filtered, genome = BSgenome.Hsapiens.UCSC.hg19)

计算偏差

  • computeDeviations()函数返回一个包含两个assay的SummarizedExperiment。
  • 第一个矩阵可以通过deviations(dev)或assays(dev)$deviations访问。将给出每个单元格或样本（列）的每组peak（行）的可访问性的偏差校正“偏差”。该指标表示peak组的可及性与基于细胞/样品中相同染色质可及性谱的预期的可及性的程度，并通过与 GC 和平均可及性匹配的一组背景peak进行归一化。
  • 第二个矩阵deviationScores(dev)或assays(deviations)$z，它考虑了如果对具有相似 GC 含量和平均可达性的peak组进行随机采样，出现这种分数的可能性有多大。
xxxxxxxxxx
# 计算偏差
dev <- computeDeviations(object = counts_filtered, annotations = motif_ix)
dim(dev)

deviationScores(dev)[1:4,1:2]

背景peak

  • computeDeviations()将使用一组背景峰值来标准化偏差分数。该计算默认在内部完成且不返回 - 为了更好地控制此步骤，用户可以自己运行getBackgroundPeaks()函数并将结果传递给computeDeviations()的参数background_peaks。
  • 背景peak是与 GC 含量和平均可及性peak相似的peak。
xxxxxxxxxx
# 背景peak
bg <- getBackgroundPeaks(object = counts_filtered)
dim(bg)
xxxxxxxxxx
[1] 28596    36
  • 得到的结果为一个索引矩阵，每一列表示的是作为背景peak的索引。
  • 使用计算得到的背景peak：
xxxxxxxxxx
# 使用计算得到的背景peak
dev <- computeDeviations(object = counts_filtered, annotations = motif_ix,
                         background_peaks = bg)
