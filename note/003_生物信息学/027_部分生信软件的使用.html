scSplit

  • 软件使用：https://github.com/jon-xu/scSplit/
  • 对合并的单细胞 RNA-seq 进行无基因型的拆分，使用隐藏状态模型来识别混合群体中遗传上不同的样本。

整体流程

1. 数据质量控制和过滤

  • 过滤条件：read质量值低于10，未匹配的片段，存在二次匹配，未通过过滤，PCR或光学重复，存在补充比对。
# 例如
samtools view -S -b -q 10 -F 3844 processed.bam > filtered.bam
  • 如有必要，使用 UMI-tools 中的 rmdup 等工具删除基于 UMI 的重复read。
  • 使用 samtools 中的 sort、index 命令对 BAM 文件进行排序和索引。

2. 寻找单核苷酸变体

  • 使用freebayes v1.2从第一步处理后的混合样本BAM文件中调用SNV，设置freebayes的参数，使得没有插入和删除（indels），也没有多核苷酸多态性（MNP）或复杂事件将被捕获，将最小等位基因计数设置为 2，并将最小碱基质量设置为 1。
xxxxxxxxxx
freebayes -f <reference.fa> -iXu -C 2 -q 1 Filtered_rmdup_sorted.bam > snv.vcf
  • 此步骤可能需要很长时间（如果不使用并行处理，则长达 30 小时）。为了加快调用过程，用户可以按染色体拆分BAM，分别调用SNV，然后合并vcf文件。
  • 应使用 bcftools 进一步过滤输出的 VCF 文件，以便仅保留质量得分大于 30 的 SNV。
xxxxxxxxxx
# 例如
bcftools filter -i '%QUAL>30' snv.vcf -o filtered.vcf

3. 构建等位基因计数矩阵

  • 运行“scSplit count”并获取两个 .csv 文件（“ref_filtered.csv”和“alt_filtered.csv”）作为输出。

    -c, --com, common SNVs

    -a, --alt, output Alt count matrix

# 例如
scSplit count -v filtered.vcf -i filtered_rmdup_sorted.bam -b barcodes.tsv -r ref_filtered.csv -a alt_filtered.csv -o .

  • 请使用 -c/--com 参数指定 scSplit count 中的常见 SNV，请确保您的常见 SNV 列表没有标题行，并且请确保常见 SNV 文件中的染色体格式与您的数据中的一致，例如，如果您的数据使用“chr1”而不是“1”来表示染色体，则需要在运行 scSplit 之前在通用 SNV 文件的每一行的开头添加“chr”。

  • 在构建计数矩阵时，将检查每个等位基因的基因型，并且仅保留那些杂合的、因此信息丰富的 SNV。这是通过检查 VCF 文件中的 GL/GP/PL/GT 字段来实现的，如果存在，则仅使用按顺序排列的字段。

  • 如果 SNV 和/或细胞的数量较多，此步骤可能会消耗内存。作为指导，为 60,000 个 SNV 和 10,000 个单元构建矩阵可能需要超过 30GB RAM 才能运行，请留出足够的 RAM 资源来运行脚本。

  • 此步骤的典型运行时间约为一小时，具体取决于数据的性质和分配的资源。

  • 此步骤后过滤的 SNV 的典型数量通常在 10,000 到 30,000 之间。

  • 如果此步骤失败，请检查：

      1. BAM 文件中的条形码标签是否为“CB” - 如果不是，则需要使用 -t/--tag 指定；
      2. 您是否正在研究混合样本 VCF，而不是单个基因型的简单合并？
      3. 是否使用了正确的白名单条形码文件？白名单应该是测序结果中可信的条形码，而不是测序方案的整个条形码库。

Vireo

  • github：https://github.com/single-cell-genetics/vireo
  • 使用手册：https://vireosnp.readthedocs.io/en/latest/index.html
  • 一个使用贝叶斯的软件，用于拆分单细胞的混样数据，有基因型的参考或没有。
  • Vireo 主要设计用于通过建模将细胞分离为供体 表达的等位基因。它支持多种供体基因型设置（从 完全缺失、部分缺失、完全观察到）
  • 作为一般的通过等位基因比率进行细胞聚类的方法（相当于基因分型）， Vireo 适用于供体细胞拆分之外的更多设置，包括 体细胞克隆的重建，请参阅 vireoSNP_clones.ipynb 例如 线粒体突变。

手册

  • 拆分细胞需要两个读数或 UMI 计数矩阵：AD(替换等位基因)和DP(深度)。这两个矩阵可以是 通过对每个细胞中的一系列变体进行基因分型来获得。我们提供指南 对于细胞基因分型，推荐使用cellSNP-lite 由我们开发。
  • 获得每个细胞的基因型（例如，以 VCF 格式）或两个 稀疏矩阵AD和DP，我们可以应用Vireo进行细胞拆分。

拆分不同的样本

  • 默认情况下，Vireo 在没有任何已知基因型信息的情况下工作对于混样样品。然而，如果这些样本的任何基因型是已知的或可以是 例如通过bulk RNA-seq、exome-seq获得的，添加它们仍然有用，而不是 不仅允许我们将解卷积样本与其身份对齐，而且还可以 有利于双峰识别，特别是如果覆盖范围或负载 每个样本的细胞数较低。

  • 四种策略 对 scRNA-seq 数据进行样本拆分：

  • 没有任何基因型：

    xxxxxxxxxx
    vireo -c $CELL_DATA -N $n_donor -o $OUT_DIR

  • 所有样本的基因型（基因标签：GT、GP 或 PL；默认为 PL，请 选择已存在的）：

    xxxxxxxxxx
    vireo -c $CELL_DATA -d $DONOR_GT_FILE -o $OUT_DIR

    为了有效加载DONOR VCF 文件，我们建议将其子集化

    xxxxxxxxxx
    bcftools view donor.vcf.gz -R cellSNP.cells.vcf.gz -Oz -o sub.vcf.gz

  • 部分样本的基因型（n_donor 大于 DONOR_GT_FILE）：

    xxxxxxxxxx
    vireo -c $CELL_DATA -d $DONOR_GT_FILE -o $OUT_DIR -N $n_donor

  • 有基因型但不可信（或仅针对 SNP 子集）：

    xxxxxxxxxx
    vireo -c $CELL_DATA -d $DONOR_GT_FILE -o $OUT_DIR --forceLearnGT

细胞数据的格式

  • 支持3种格式的细胞数据：

    • cellSNP 输出文件夹，其中包含变体信息和稀疏矩阵的 VCF AD 和 DP
    • Vartrix 输出三个或四个文件：alt.mtx、ref.mtx、barcodes.tsv[、SNP.vcf.gz]
    • 标准 VCF 文件，包含单元格变体

  • 歧视性变体：给定一组可获得估计基因型的变体，Vireo 软件实现了一种启发式方法来定义一组最小且信息丰富的 歧视性变体。这组变体可用于执行基于 qPCR 的 基因分型或其他靶向基因分型方法。简单来说，这个算法 优先考虑在分割样本时具有最大信息增益的变体。

    xxxxxxxxxx

基因分型

  • 对每个细胞上的常见变异进行基因分型（或堆积）列表是一个预先步骤 用于使用 Vireo 对它们进行拆分。
  • 细胞基因分型可分为以下两个子步骤， 不同的情况，可能需要定制策略以确保 高质量基因分型数据以分离细胞。

细胞基因分型的推荐策略

  • 对于人类或基因分型物种：变体列表（给定）+ cellsnp-lite（正在输入）。
  • 对于没有已知常见变体的物种：cellsnp-lite（使用模式 2b 调用 &使用模式 1a) 打字。 freebayes 是 cellsnp-lite 模式 2b 的替代方案 用于杂合 SNP 的调用。
  • cellSNP 最初是基于 pysam 用 Python 开发的，即 对于几千个细胞来说很方便，但成为计算瓶颈 对于大量细胞。因此，我们重新实现了它 cellsnp-lite 采用 C/C++ 语言，速度提高约 5 倍，内存减少约 50 倍。

1. 确定候选 SNP

  • 识别候选 SNP 的方法有多种，每种方法都有独特的特性 并且可能适合不同的情况或物种。在这里，我们列出了两种常见的 策略。

    • 1)：使用常见变体：最好的方法是通过基因分型对每个合并样本进行基因分型 阵列或外显子组/基因组/RNA 测序（或其他组学数据），并使用此列表 区分变体作为对每个细胞进行基因分型的候选者。然而，这可以 成本高昂，并且在大多数情况下没有必要。

      对于人类来说，已经确定了非常全面的常见变体列表 通过国际努力，例如 10000 基因组计划 和 gnomAD 每个细胞上有数百万个常见的 SNP 进行基因分型。好处包括 减少混杂因素，例如由 RNA 编辑引起的混杂因素。我们通常建议这样做，如果 对于人类，我们提供了一些预处理的 SNP 列表。

    • 2)：从 scRNA-seq 中调用变体：除了人类之外，大多数物种可能没有明确定义的常见变异， 因此，最好的方法是直接从混合的 scRNA-seq 中调用变体。

      软件包freebayes是一个经常的选择，它旨在寻找小的 多态性，特别是 SNP、插入缺失、MNP 和小于 短读长测序比对的长度。重要的是，freebayes 有 一组用于过滤读取和变体的选项。

      我们还推荐我们开发的替代方法cellsnp-lite。 它的模式 2b 具有类似的功能，可以堆积整个基因组并识别 合并样本中的杂合变异。具有很强的可比性 精度达到 freebayes 和 bcftools mpileup 并实现 5-10 倍的加速。

2. 对每个细胞进行基因分型

  • 一旦找到候选变体列表，就可以更直接地 对每个细胞进行基因分型。

  • 我们提供了三种常用的方法，推荐 我们的cellsnp-lite与 Vireo 无缝连接。

    • bcftools / samtools 中的著名的mpileup 通常是一个不错的选择。然而， 这可能会很慢，因为它不允许并行计算，而且它也不允许 支持许多细胞的池 BAM 文件中的细胞条形码和 UMI 标签。
    • 这一限制促使我们开发cellSNP，一种 pysam 包裹物（现在 cellsnp-lite（C/C++ 中））来堆积 每个细胞中的变异。好处包括并行计算、采用单元格 条形码标签并支持UMI。
    • 或者，vartrix 也是在 10x Genomics 中对细胞进行基因分型的选项 数据。

Souporcell

  • 软件使用：https://github.com/wheaton5/souporcell

  • souporcell 是一种按个体对混合基因型 scRNAseq 实验进行聚类的方法。

  • 输入只是 possorted_genome_bam.bam，以及作为 cellranger 输出的 Barcodes.tsv。 souporcell 由 6 个步骤组成，前 3 个步骤使用外部工具，最后一个步骤使用此处提供的代码。

    • Remapping (minimap2)
    • Calling candidate variants (freebayes)
    • Cell allele counting (vartrix)
    • Clustering cells by genotype (souporcell.py)
    • Calling doublets (troublet)
    • Calling cluster genotypes and inferring amount of ambient RNA (consensus.py)

  • 下载singularity镜像（1.3gb）（singularity与docker类似，但对集群来说是安全的）

    xxxxxxxxxx
    singularity pull shub://wheaton5/souporcell

  • 要求：singularity >= 3.0

    xxxxxxxxxx
    which singularity
    singularity --version

  • 您现在应该能够通过singularity容器运行 souporcell_pipeline.py。 Singularity 会自动挂载当前工作目录以及运行它的下游目录，否则您需要手动挂载这些目录。因此，您可以从所有输入上游的目录运行它。输入文件是 cellranger bam、cellranger 条形码文件和参考 fasta。 cellranger bam 位于 cellranger outs 目录中，名为 possorted_genome_bam.bam。条形码文件位于 cellranger outs/filtered_gene_bc_matrices/

  • 设置 --group-list 为条形码列表（即 10X 中的barcodes.tsv）以加快速度，并且仅对其他方法调用的单元进行多路分解。

介绍

demuxlet 是一款软件工具，用于在通过条形码单细胞测序汇集多个样本时，对样本身份进行解卷积并识别多重样本。 demuxlet 采用 (1) 标准 10x 测序平台或任何其他带条形码的单细胞 RNA-seq 生成的 SAM/BAM/CRAM 文件（具有适当的--tag-UMI 和 --tag-group) 选项 (2) 包含基因型 (GT)、后验概率 (GP) 或基因型可能性 (GL) 的 VCF/BCF 文件，用于将每个条形码分配给特定样本（或VCF 文件中的一对样本）。

  • 注意 目前不支持 htslib 1.11 - 请使用早期版本（例如 1.10.x）

输出文件的解释

  • demuxlet 生成多个输出文件，例如 [prefix].best, [prefix].sing, [prefix].sing2和可选的[prefix].pair（带有 --write-pair参数），每个文件包含以下信息。

    • [prefix].best文件包含样本身份的最佳猜测，并具有详细的统计信息，可以达到最佳猜测
    • [prefix].sing 文件包含每个细胞与每个可能样本的匹配统计信息。
    • [prefix].sing2 文件包含与前一个文件类似的统计信息，但生成的目的是为了对[prefix].pair 结果进行健全性检查。
    • [prefix].pair 文件包含将每个单元与双细胞的每种可能配置相匹配的统计信息。

  • [prefix].best 文件包含以下 22 列。

    • BARCODE - 在此行中分配的单元格的单元格条形码

    • RD.TOTL - 每个液滴与变异位点重叠的读取总数。

    • RD.PASS - 通过质量阈值的读取总数，例如映射质量、碱基质量。

    • RD.UNIQ - 通过质量阈值的 UMI 总数。如果在单个变体中多次观察到 UMI，则不会对其进行更多计数。如果在多个变体中观察到 UMI，则它将被视为不同。

    • N.SNP - 与液滴中任何读数重叠的变体总数。

    • BEST - 样品 ID 的最佳分配。

      • 对于单线，SNG-
      • 对于双联体，DBL---
      • 对于不明确的液滴，AMB---<双联ID1/ID2>）

    • SNG.1st-样本ID的最佳单曲分配

    • SNG.LLK1 - 日志（SNG.1ST 中的 ID 是正确分配的可能性）

    • SNG.2ND - 样品 ID 的下一个最佳单峰分配

    • SNG.LLK2 - 日志（SNG.2ND 中的 ID 是正确分配的可能性）

    • SNG.LLK0 - 仅等位基因频率的对数似然

    • DBL.1ST - 如果分配是双峰，则最有可能包含的样本 ID

    • DBL.2ND - 如果分配是双峰，则下一个最可能包含的样本 ID

    • ALPHA - % 混合物比例

    • LLK12 - 对数（ID 为双峰的可能性）

    • LLK1 - 日志（DBL.1ST 中的 ID 是正确单态分配的可能性）

    • LLK2 - 对数（DBL.2ND 中的 ID 是正确单态分配的可能性）

    • LLK10 - 对数（DBL.1ST 的 ID 是双联体之一的可能性，而另一个双联体身份仅根据等位基因频率计算）

    • LLK20 - 对数（DBL.2ND 的 ID 是双联体之一的可能性，而另一个双联体身份仅根据等位基因频率计算）

    • LLK00 - 对数（液滴为双联体的可能性，但两个身份仅根据等位基因频率计算）

    • PRB.DBL - 双峰分配的后验概率

    • PRB.SNG1 - 排除所有可能的双峰时单峰分配的后验概率

freebayes

xxxxxxxxxx
usage: freebayes [OPTION] ... [BAM FILE] ... 


Bayesian haplotype-based polymorphism discovery.


citation: Erik Garrison, Gabor Marth
          "Haplotype-based variant detection from short-read sequencing"
          arXiv:1207.3907 (http://arxiv.org/abs/1207.3907)


overview:


    To call variants from aligned short-read sequencing data, supply BAM files and
    a reference.  FreeBayes will provide VCF output on standard out describing SNPs,
    indels, and complex variants in samples in the input alignments.


    By default, FreeBayes will consider variants supported by at least 2
    observations in a single sample (-C) and also by at least 20% of the reads from
    a single sample (-F).  These settings are suitable to low to high depth
    sequencing in haploid and diploid samples, but users working with polyploid or
    pooled samples may wish to adjust them depending on the characteristics of
    their sequencing data.


    FreeBayes is capable of calling variant haplotypes shorter than a read length
    where multiple polymorphisms segregate on the same read.  The maximum distance
    between polymorphisms phased in this way is determined by the
    --max-complex-gap, which defaults to 3bp.  In practice, this can comfortably be
    set to half the read length.


    Ploidy may be set to any level (-p), but by default all samples are assumed to
    be diploid.  FreeBayes can model per-sample and per-region variation in
    copy-number (-A) using a copy-number variation map.


    FreeBayes can act as a frequency-based pooled caller and describe variants
    and haplotypes in terms of observation frequency rather than called genotypes.
    To do so, use --pooled-continuous and set input filters to a suitable level.
    Allele observation counts will be described by AO and RO fields in the VCF output.




examples:


    # call variants assuming a diploid sample
    freebayes -f ref.fa aln.bam >var.vcf


    # call variants assuming a diploid sample, providing gVCF output
    freebayes -f ref.fa --gvcf aln.bam >var.gvcf


    # require at least 5 supporting observations to consider a variant
    freebayes -f ref.fa -C 5 aln.bam >var.vcf


    # discard alignments overlapping positions where total read depth is greater than 200
    freebayes -f ref.fa -g 200 aln.bam >var.vcf


    # use a different ploidy
    freebayes -f ref.fa -p 4 aln.bam >var.vcf


    # assume a pooled sample with a known number of genome copies
    freebayes -f ref.fa -p 20 --pooled-discrete aln.bam >var.vcf


    # generate frequency-based calls for all variants passing input thresholds
    freebayes -f ref.fa -F 0.01 -C 1 --pooled-continuous aln.bam >var.vcf


    # use an input VCF (bgzipped + tabix indexed) to force calls at particular alleles
    freebayes -f ref.fa -@ in.vcf.gz aln.bam >var.vcf


    # generate long haplotype calls over known variants
    freebayes -f ref.fa --haplotype-basis-alleles in.vcf.gz \ 
                        --haplotype-length 50 aln.bam


    # naive variant calling: simply annotate observation counts of SNPs and indels
    freebayes -f ref.fa --haplotype-length 0 --min-alternate-count 1 \ 
        --min-alternate-fraction 0 --pooled-continuous --report-monomorphic aln.bam >var.vcf

   -h --help       Prints this help dialog.
   --version       Prints the release number and the git commit id.


input:


   -b --bam FILE   Add FILE to the set of BAM files to be analyzed.

                   An index file (FILE.fai) will be created if none exists.
                   If neither --targets nor --region are specified, FreeBayes
                   will analyze every position in this reference.

   -r --region <chrom>:<start_position>-<end_position>

                   Equivalent to -m 30 -q 20 -R 0 -S 0
   -F --min-alternate-fraction N
                   Require at least this fraction of observations supporting
                   an alternate allele within a single individual in the
                   in order to evaluate the position.  default: 0.05
   -C --min-alternate-count N
                   Require at least this count of observations supporting
                   an alternate allele within a single individual in order
                   to evaluate the position.  default: 2
   -3 --min-alternate-qsum N
                   Require at least this sum of quality of observations supporting
                   an alternate allele within a single individual in order
                   to evaluate the position.  default: 0
   -G --min-alternate-total N
                   Require at least this count of observations supporting
                   an alternate allele within the total population in order
                   to use the allele in analysis.  default: 1
   --min-coverage N
                   Require at least this coverage to process a site. default: 0
   --limit-coverage N
                   Downsample per-sample coverage to this level if greater than this coverage.
                   default: no limit
   -g --skip-coverage N
                   Skip processing of alignments overlapping positions with coverage >N.
                   This filters sites above this coverage, but will also reduce data nearby.
                   default: no limit
   --trim-complex-tail
                   Trim complex tails.


population priors:


   -k --no-population-priors
                   Equivalent to --pooled-discrete --hwe-priors-off and removal of
                   Ewens Sampling Formula component of priors.


mappability priors:


   -w --hwe-priors-off
                   Disable estimation of the probability of the combination
                   arising under HWE given the allele frequency as estimated
                   by observation frequency.
   -V --binomial-obs-priors-off
                   Disable incorporation of prior expectations about observations.
                   Uses read placement probability, strand balance probability,
                   and read position (5'-3') probability.
   -a --allele-balance-priors-off
                   Disable use of aggregate probability of observation balance between alleles
                   as a component of the priors.


genotype likelihoods:


   --observation-bias FILE
                   Read length-dependent allele observation biases from FILE.
                   The format is [length] [alignment efficiency relative to reference]
                   where the efficiency is 1 if there is no relative observation bias.
   --base-quality-cap Q
                   Limit estimated observation quality by capping base quality at Q.
   --prob-contamination F
                   An estimate of contamination to use for all samples.  default: 10e-9
   --legacy-gls    Use legacy (polybayes equivalent) genotype likelihood calculations
   --contamination-estimates FILE
                   A file containing per-sample estimates of contamination, such as
                   those generated by VerifyBamID.  The format should be:
                       sample p(read=R|genotype=AR) p(read=A|genotype=AA)
                   Sample '*' can be used to set default contamination estimates.


algorithmic features:


   --report-genotype-likelihood-max
                   Report genotypes using the maximum-likelihood estimate provided
                   from genotype likelihoods.
   -B --genotyping-max-iterations N
                   Iterate no more than N times during genotyping step. default: 1000.
   --genotyping-max-banddepth N
                   Integrate no deeper than the Nth best genotype by likelihood when
                   genotyping. default: 6.
   -W --posterior-integration-limits N,M
                   Integrate all genotype combinations in our posterior space
                   which include no more than N samples with their Mth best
                   data likelihood. default: 1,3.
   -N --exclude-unobserved-genotypes
                   Skip sample genotypings for which the sample has no supporting reads.
   -S --genotype-variant-threshold N
                   Limit posterior integration to samples where the second-best
                   genotype likelihood is no more than log(N) from the highest
                   genotype likelihood for the sample.  default: ~unbounded
   -j --use-mapping-quality
                   Use mapping quality of alleles when calculating data likelihoods.
   -H --harmonic-indel-quality
                   Use a weighted sum of base qualities around an indel, scaled by the
                   distance from the indel.  By default use a minimum BQ in flanking sequence.
   -D --read-dependence-factor N
                   Incorporate non-independence of reads by scaling successive
                   observations by this factor during data likelihood
                   calculations.  default: 0.9
   -= --genotype-qualities
                   Calculate the marginal probability of genotypes and report as GQ in
                   each sample field in the VCF output.
