测序原理

  • illumina测序的优势：

    • 可逆阻断终止技术
    • 边合成边测序
    • 双末端测序

  • 样品要求：

    • 最好为单倍体
    • 达到DNA纯度要求OD值
    • 样品未降解
    • 样品量满足建库要求

建库测序

  • 文库：就是DNA片段的一个集合，将测序片段打断之后就构成了一个DNA文库。

  • 构建文库进行随机打断，1000一下为小片段文库，此时的长度指的是峰值出的长度。

  • 在3’端添加A碱基（便于连接引物），之后连接上引物，添加index标签，用于区分不同的物种。添加adapter接头（P5、P7），此接头与测序芯片的接头互补配对。

  • 测序芯片：flowcell

cluster

上机测序

大片段文库

  • 大片段文库一般是指文库片段长度大于1K的片段，1k 以下普遍称为小片段文库。

  • 大片段测序的目的：获得reads之间的物理距离关系。

  • 问题：

    • 无法PCR太长的片段
    • 只测序很短的序列，合成大片段浪费

  • 小片段文库为pairend，大片段文库为matepair

测序原理FAQ

  • 为什么不能完整的测序基因组：

    • 提取出来的基因组不是完整的一条，断裂成很多片段
    • 测序当中使用了PCR，限制了读长

fastq文件

  • fasta的质量文件与序列文件的碱基一一对应，值越大，质量越高。

  • fastq为上面两个文件的合并。

  • 质量表示有两种：

    • ASCII码 - 64（Phred64）：+64一方面的原因是@的ASCII为64，小于33的ASCII是一些非字符的。辨别，如果都是字母，一般是-64。
    • ASCII码 - 33（Phred33）：辨别，如果有@等其他较为奇怪的字符，一般是-33。

测序饱和度评估

  • 饱和度用于推出测序所需的数据量。

  • 测序不饱和的影响：

    • 对于DNA基因组测序来说，如果测序不饱和影响序列拼接
    • RNAseg定量不准
    • 宏基因组不能准确反映物种的组成

测序数据质控

  • 数据质控的指标：

    • 碱基含量分布
    • 碱基质量分布

  • 测序数据与基因组GC含量一致：

    • GC含量表示测序数据中ATCG四种碱基分布
    • ATCG含量恒定，与基因组碱基分布一致
    • 测序随机均匀，ATCG同时倍增

短序列比对

  • 短序列比对就是将这些测序的reads重新定位到基因组上，这个过程也叫做回贴或者叫做mapping。

  • 数据集：

    • 基因组(或者基因集 )，fasta格式、核酸，可以为全基因组，基因集，rRNA等
    • 测序短reads (PE或者SE)，fastq格式

  • illumina测序数据特点：

    • 测序覆盖全基因组
    • 测序数据读长短
    • 测序数据具有一定的错误率
    • 测序数据深度高
    • 测序数据具有pair-end关系

  • PE与SE比对：

    • 两条reads同时比对到同一序列(pairend 比对
    • 只有一条reads比对上目标序列 (single比对 )
    • 两条reads比对到不同序列 single比对)
    • 两条reads比对超出insertsize范围 (single比对 )

短序列比对的作用

  • 与自身基因组比对：

    • 计算每个位点覆盖深度
    • 计算参考序列覆盖比率

  • 与参考基因组比对：

    • RNA测序计算基因表达量
    • 宏基因组测序计算不同生物的丰度
    • 变异检测

  • reads利用率为比对到目标序列的reads与reads总数的比例。 reads利用率=比对上的reads数/总reads数

  • 覆盖深度，Coverage depth ，也简称为覆盖度，也叫乘数是指每个碱基被测序的平均次数，是用来衡量测序量的首要参数。

  • 覆盖比率，Coverage ratio，也简称覆盖率，指被测序到的碱基占全基因组大小的比率。

短序列比对常用软件

  • 常用算法：

    • 空位种子片段索引法
    • Burrows Wheeler转换法
    • Smith-Waterman动态规划算法

  • 数据：

    • 基因组(或者基因集 )，fasta格式、核酸，可以为全基因组，基因集，rRNA等
    • 测序短reads (PE或者SE)，fastq格式

  • 过程：

    1. 对目标序列fasta文件建立索引
    2. 进行短序列比对

  • 软件：

    • BWA：BWA的名字来源于使用bwt算法的aligner工具。

      1. 对参考序列建立索引（bwa index）
      2. 对每条reads建立索引文件（bwa aln）
      3. sample比对（bwa sample）

    • SOAP：

      1. 建立索引（2bwt-builder）
      2. soap比对

    • bowtie：bowtie应用于RNAseg分析

      • bowtie+Tophat+Cufflinks

  • 短序列比对注意事项：

    • 设置insertsize
    • 控制错配数
    • 控制容gap数
    • 监控内存消耗
    • 多线程提高比对效率

sam文件格式

  • SAM是一种序列比对格式标准，它的全称是(The Sequence Alignment/Map(SAM) format )，由sanger制定。是以TAB为分割符的文本格式。主要应用于测序序列mapping到基因组上的结果表示，当然也可以表示任意的多重比对结果。

  • bam为sam文件的二进制压缩格式。

  • sam可以直接打开查看，而bam文件没有办法，需要借助samtools等工具。

  • sam文件分为两部分：

    • 注释信息：可有可无，以@开头

    • 比对结果：

    • 如果flag不是以上的数字，则是以上数字的组合的和。

  • 插入片段insertsize大小，也就是文库片段的长度同样也是两条测序reads的物理距离

  • 比较基因差异表达的标准：

    • fold change（FC）
    • 通过FDR校正

短序列比对FAQ

  • 建立索引错误：

    • 目标序列不能太短，否则无法建立索引；
    • 序列文件中不要有回车符；
    • 选择正确的bwa index 选项；

  • 短序列比对的资源消耗：使用bam来减少数据存储

  • 如何提高比对的效率：

    • 完善软件的算法；
    • 提高计算机硬件资源；
    • 比对前要对数据进行处理；
    • 通过数据拆分合并提高比对效率；

  • matepair文库：

    • soap比对中要加-R选项
    • bowtie比对中要加--fr或者--rf，--ff等选项来指明read的方向

  • 短序列比对与长序列比对之间的差别：

    • 长序列是比对多少的问题，短系列是比对有无的问题
    • 长序列可以容许更多的gap和错配
    • 亲缘关系太远无法使用短序列比对结果不好

序列拼接

  • 高通量测序的两条路线：短序列比对和序列拼接。

  • 序列拼接存在的问题：

    • 两条序列的方向
    • overlap的大小
    • overlap之间存在错配
    • 一条序列与多条序列之间存在overlap
    • 连接之后是否可以继续连接

  • 序列拼接相关名词：

    • reads
    • pairend（小片段）与matepair（大片段）
    • insertsize
    • kmer：将reads切割成的小片段
    • contig：kmer/reads之间连接而成的长片段
    • scaffold：contig之间的连接而成

pregraph

  • 序列拼接可用数据：

    • 两条pairend关系reads
    • reads之间具有overlap
    • reads之间具有pairend关系

  • 短序列拼接的步骤：

    • 构图
    • 构建contig
    • 构建scaffold (包括map)
    • 补洞

  • 为什么要切kmer：因为测序中存在错误。去掉kmer频数为1的。

构建contig

  • 在去掉低频率的kmer后，得到一个相对稳定的kmer集合。
  • 两个kmer之间必须有n-1个overlap才可以进行连接。

  • 常见名词：

    • 图中每个kmer为一个node。
    • node之间的连接为一个edge。
    • 无法继续连接下去的端点称为一个tips。
    • kmer的连接出现分支，之后分支又连接到一起称为bubble。

  • 简化De bruijin图

构建scaffold

  • 通过reads之间的pairend(或matepair)关系，将contig连接成更长的序列我们称为scaffold。
  • contig之间没有overlap的关系。
  • 连接为scaffold之后还是有很多片段
  • 接下来可以使用matepair确定scaffold之间的位置关系。

  • 总的来说，拼接策略：

    • 首先利用小片段文库
    • 逐步加入大片段文库

基因组污染分析

  • 基因组污染特征：

    • 基因组明显偏大
    • 序列丰度不同
    • GC异常

  • 序列唯一性：序列越长唯一性越高

  • 污染鉴定：

    • 与NCBI比对进行鉴定
    • 预测16s(或18s )进行鉴定

  • 污染处理：

    • 由于序列之间存在相似性，丰度存在交叉，无法准确区分开污染序列与正常序列
    • 重新提取样品进行建库测序

RNAseg与meta序列拼接

  • DNA测序与RNAseq比较：

    • DNA一般为全基因组测序，而RNA测序的是转录出来的转录本，都是独立断开的片段;
    • DNA测序一般是均匀测序，基因组上的区域被均匀覆盖，而RNA由于存在表达丰度的差异
    • DNA全基因组测序中存在很多重复序列，干扰序列的拼接，而转录组中这个问题影响会小一些

  • RNA拼接注意事项：

    • 拼接结果中获取Unigene
    • 拼接时要去除duplication
    • 表达定量时不能去除duplication

  • 宏基因组拼接：宏基因组(Metagenome)，也称微生物环境基因组MicrobialEnvironmental Genome,或元基因组。

序列拼接FAQ

  • 影响拼接的因素有那些：

    • 内因：多倍体，基因组杂合，高度重复，低复杂度，GC偏差等
    • 外因：测序数据量，测序质量，文库大小，kmer大小，基因组自身，拼接软件，阈值设定

  • 如何改善拼接效果：

    • 覆盖基因组所有位点；
    • 重新调整insertsize；
    • 去除insertsize偏差大的pairend reads；
    • 调整kmer大小以及软件选项参数阈值；
    • 混合拼接

  • 为什么不用短reads直接overlap拼接：

    • reads中存在错误率:
    • 通过kmer去除包含reads中错误的位点

  • 如何选择kmer的大小：

    • reads准确度越高，选择大kmer较好
    • reads错误率高，选择小kmer，reads利用率高
    • 基因组本身特点，重复率，测序深度等因素，都会对kmer取值造成一定影响

  • 为什么kmer只能是奇数：考虑回文序列

基因组分析介绍

基因预测

  • 基因预测方法：

    • 利用软件对物种基因组直接进行预测
    • 通过同源序列比对，和已知近源物种基因集进行比对，将同源比对区筛选出来，作为基因。

  • 方法比较：

    • 从头预测：不需要同源参考基因序列直接进行预测，非常方便，适合于新发现的物种，没有很多已知的基因信息；
    • 基于同源基因的序列比对：找出的基因则更加准确，但是如果没有同源序列，或者同源区不含有某个基因的话，就会漏掉一些基因。
  • 宏基因组基因预测：宏基因组中包含原核生物、真核生物、病毒等，基因预测难度大。

序列比对概述

  • 同源(homology): 是指来自于同一个祖先，两个物种从同一个祖先分化后，与不同的环境发生相互作用，其相应的DNA序列将各自发生一些替换或者插入缺失突变，也就是说序列不在精确相同。

  • 相似性(similarity): 是指字符串的相似，并不具有生物学意义。

  • 同源序列一般是相似的，相似序列不一定是同源的。

  • 序列两两比对：

    • 两个碱基类型完全一致，没有发生变化。
    • 碱基发生了变化，这种变化包括替换、插入和删除。

  • 比对算法：

    • Needleman算法
    • Smith-Waterman算法
    • 启发式联配算法等

局部比对与全局比对

  • 全局比对是用来衡量两条序列整体的相似性，满足整体相似性最大化。
  • 局部比对则不同，两条亲缘关系较远的DNA或氨基酸可能只在一些片段上相似，这就需要找到这些相似性的片段，和其相应的匹配方式。

多序列比对

  • 多序列比对(Multiple sequence alignment;MSA )，是对三个以上的生物学序列，如蛋白质序列、DNA序列或RNA序列所作的序列比对。

  • 构建多序列比对模型方法：

    • 手工比对方法
    • 同步法
    • 渐进法

  • 多序列比对特点：

    • 多序列比对一般都是将所有的序列放在同一个文件中
    • 每一个样品都会对整体造成很大的影响
    • 序列之间要具有同源关系，并且具有同一方向

变异检测概述

  • 遗传就是子代在这个连续系统中重复亲代的特性和特征的现象。

  • 基因组世代延绵的长期发展过程中，随着时间和空间的变化。结构改变了的基因使生物体发育不同于改变前的性状，于是出现了变异。

  • 测序：

    • Denovo测序：如果一个基因组第一次被测序出来，我们一般称为Denovo测序，主要是需要拼接其基因组
    • 重测序：已有基因组被发表出来之后，重新测序新的数据，可以不需要护接，而直接用测序的reads进行短序列比对分析，称为重测序分析。在文献中常常会看到Re-sequencing这个词。其实重测序本质就是找变异。

  • 变异检测的意义：

    • 引起基因组的变化，进而导致基因型或者调控区的变化
    • 根据基因型的变化推测与表型变化之间的关系
    • 构建系统发育树，进行系统发育分析等

SNP检测

  • SNP (single nucletide polymorphsysm )，即单核昔酸的多态性是指等位基因发生了突变致使不同等位型的出现。

  • SNP：

    • 纯合SNP：纯合SNP是指两个等位基因都发生了同样的突变
    • 杂合SNP：杂合SNP是指两个等位基因中发生了不同的突变

  • SNP的检测中一定要使同源序列的比对才有意义。

  • SNP检测方法：

    • 利用测序数据与参考序列进行短序列比对
    • 利用序列拼接经过与参考序列进行比对

SV检测

  • 基因组结构变异SV是Structural Variation的简称。广义上来说，基因组上所有的变异都可以称为SV。而更狭义的概念，SV特指基因组大片段的插入与缺失。

CorePanGene集构建

  • Pan-genome: The global gene repertoire of a bacterial species
  • Core-genome: The pool of genes shared by all the strains of thesame bacterial species（类似于基因集的交集）
  • Dispensable genome: The pool of genes present in some but notall strains of the same bacterial species（类似于基因集的并集）

    • 去除reads支持数为1的低频突变
    • 支持SNP的reads质量值大于Q20
    • SNP位点附近3bp内没有另外的SNP
    • SNP的覆盖度要大于3
    • 在任意连续的10bp中不超过3个SNP
    • 去除趋同进化的SNP等

系统发育树构建

  • 进化树分为分子树与系统发育树。分子树是依据分子数据构建的反映分子系统发育的树。系统发育树 就是描述这一群有机体发生或进化顺序的拓朴结构。

  • 构建系统发育树方法：

    • 序列类似性比较，主要是基于氨基酸相对突变率矩阵，例如PAM250矩阵，计算不同序列差异性积分作为它们的差异性量度
    • 通过蛋白质结构比较包括刚体结构叠合和多结构特征比较等方法建立结构进化树。

  • 多序列比对构建系统发育树：

    • coregene
    • MLST
    • MLVA
    • 全基因组SNP

  • Coregene与SNP方法比较：

    • coregene 包括SNP与InDel的差异，但是不包括基因间区
    • 全基因组SNP包含基因组与间区SNP变化，但是不包括插入与缺失信息

  • 构建系统发育树的常用方法：

    • 独立元素法：所谓独立元素法是指进化树的拓扑形状是由序列上的每个碱基/氨基酸的状态决定的。最大简约性法(Maximum Parsimony methods)和最大可能性法(Maximum Likelihood methods )
    • 距离依靠法：而距离依靠法是指进化树的拓扑形状由两两序列的进化距离决定的。除权配对法(UPGMAM)和邻位相连法(Neighbor-joining )。

  • 系统发育树评估：Bootstrap，即自展值，是用来检验计算的进化树分支可信度的。简单地讲就是把序列的位点都重排，重排后的序列再用相同的办法构树，如果原来树的分枝在重排后构的树中也出现了，就给这个分枝打上一分如果没出现就给0分，这样经过你给定的repetitions次(至少1000次)重排构树打分后，每个分枝就都得出分值，计算机会给你换算成bootstrap values值。

  • Bootstrap values 即步长值，是指在你选择的遗传距离算法中软件根据所比对序列得到结果比如 bootstrap value设置为1000，即软件构建了，在每个节点上显示的bootstrap value 即指在这相应的1000”棵树1000次建树过程中，有相应的次数的频率这个分枝内的几株菌或几段序列在进化速度上相似。
