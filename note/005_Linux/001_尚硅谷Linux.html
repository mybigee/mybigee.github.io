6.目录结构

  1. 在 Linux 世界里，一切皆文件。

  3. 常用目录：

    • /bin：binary的简称，存放常用命令。
    • /sbin：super user，存放系统管理员使用的系统程序。
    • /home：存放所有普通用户的主目录，每一个用户都有一个自己的主目录。
    • /root：系统管理员目录。
    • /boot：存放启动Linux的核心文件，包括一些链接文件和镜像文件。
    • /proc：为虚拟目录，是系统内存的映射，访问该目录获取系统信息。
    • /srv：service的缩写，该目录存放一些服务启动之后的数据。
    • /sys：系统文件夹。
    • /tmp：存放临时文件。
    • /dev：设备管理文件，将所有硬件以文件的方式存储。
    • /media：系统识别的一些设备，如U盘、光驱等，Linux会把识别的设备挂载到该目录下。
    • /mnt：让用户临时挂载别的文件系统。
    • /opt：主机额外安装软件的目录。
    • /usr/local：另一个给主机额外安装软件的目录，一般通过编译源码的方式安装软件。
    • /var：习惯将经常修改的目录放在该文件目录下，如日志文件。
    • /selinux：安全子系统，可以控制程序只访问特定文件。

14.用户管理

  • 一个用户至少属于一个组。创建用户时没有指定用户组，会创建一个同名的组，并将该用户放于该组中。
  • 删除用户([root@myLinux home]# userdel lj)时出现userdel: user lj is currently used by process 2471，可以用Ctrl + d（退出当前用户），直到要删除的用户已经退出登陆，之后就可以删除该用户了。

  1. 添加用户：useradd 用户名

  2. 通过指定目录创建用户：useradd -d 路径 用户名（创建时该目录不要存在，存在会报错）

  3. 添加或修改密码：passwd 用户名

  4. 只删除用户：userdel 用户名

  5. 删除用户同时删除该用户的家目录：userdel -r 用户名

  6. 用户信息查询：id 用户名

  7. 切换到其他用户：su - 用户名

    切换到root时输入su -，输入exit/logout可以回到切换前的用户。高权限到低权限不需要密码，反之则需要。

  8. 查看当前用户是谁：whoami

  9. 新建用户组：groupadd 组名

  10. 删除用户组：groupdel 组名

  11. 创建用户同时添加到某个组：useradd -g 组名 用户名

  12. 修改用户的组：usermod -g 用户组 用户名

  13. 与用户相关的文件（可以使用vim打开，直接打开会显示没有权限）：

    • 所有用户的配置文件：/etc/passwd
    • 所有密码的配置文件：/etc/shadow
    • 所有用户组的配置文件：/etc/group

15.运行级别

  1. 级别

    • 0 ：关机
    • 1 ：单用户【找回丢失密码】
    • 2：多用户状态没有网络服务
    • 3：多用户状态有网络服务
    • 4：系统未使用（保留），可能以后使用
    • 5：图形界面
    • 6：系统重启

    • 查看运行级别数字：runlevel

16.单用户模式修改root密码

  1. 必须在本机操作，不能通过xshell远程操作。
  2. 原理：进入单用户模式，直接以root登录，不需要root密码
  3. 在开机引导界面按e
  4. 将出现的界面中的ro，修改为rw init=/sysroot/bin/bash
  5. 按ctrl+x执行
  6. 输入chroot /sysroot
  7. 输入passwd root修改root密码（明文修改：echo 'ad123654' | passwd --stdin root）
  8. 修改完成后运行命令 touch /.autorelabel 使SELinux生效，密码生效。
  9. 退出exit
  10. 重启reboot
  11. 修改完成

19.权限管理

  1. -rwxrw-r--

    • 第0位为文件类型：

      -：普通文件

      d：目录

      1：软连接

      c：字符设备（键盘、鼠标）

      b：块文件（硬盘）

    • 1-3为文件所有者权限

    • 4-6为文件所属用户组权限

    • 7-9为其他用户权限

    • 之后的数字：

      • 文件：表示硬连接数

      • 目录：表示子目录数(不包括文件)

    • 显示的时间为最后修改的时间

    • 显示的文件大小包括文件的字节数与一个隐藏字符，因此文件的大小为文件字节数+1。

  • w：可以修改文件，但不能删除文件，要删除文件必须有该文件所在目录的w权限

  • w：表示可以修改目录，即在文件夹内创建、删除、重命名文件夹
  • x：表示可以进入该目录

21.磁盘分区

  1. 分类：

    1. mbr 分区：

      • 最多支持四个主分区
      • 系统只能安装在主分区
      • 扩展分区要占一个主分区
      • MBR 最大只支持 2TB，但拥有最好的兼容性

    2. gpt 分区:

      • 支持无限多个主分区（但操作系统可能限制，比如 windows 下最多 128 个分区）
      • 最大支持 18EB 的大容量（1EB=1024 PB，1PB=1024 TB）
      • windows7 64 位以后支持 gpt

  2. 查看Windows的磁盘分区：搜索磁盘管理

  3. Linux分区：

    1. Linux 来说无论有几个分区，分给哪一目录使用，它归根结底就只有一个根目录，一个独立且唯一的文件结构 , Linux 中每个分区都是用来组成整个文件系统的一部分。
    2. Linux 采用了一种叫“载入”的处理方法，它的整个文件系统中包含了一整套的文件和目录， 且将一个分区和一个目录联系起来。这时要载入的一个分区将使它的存储空间在一个目录下获得。

  4. Linux硬盘：

    • Linux 硬盘分 IDE 硬盘和 SCSI 硬盘，目前基本上是 SCSI 硬盘。
    • 对于 IDE 硬盘，驱动器标识符为“hdx~”,其中“hd”表明分区所在设备的类型，这里是指 IDE 硬盘了。“x”为盘号（a 为基本盘，b 为基本从属盘，c 为辅助主盘，d 为辅助从属盘）,“~”代表分区， 前四个分区用数字 1 到 4 表示，它们是主分区或扩展分区，从 5 开始就是逻辑分区。例，hda3 表示为第一个IDE 硬盘上的第三个主分区或扩展分区,hdb2 表示为第二个IDE 硬盘上的第二个主分区或扩展分区。
    • 对于 SCSI 硬盘则标识为“sdx~”，SCSI 硬盘是用“sd”来表示分区所在设备的类型的，其余则和 IDE 硬盘的表示方法一样。

10.两个安装目录
  • /usr/local目录和/opt目录都是用来安装第三方软件的目录，所谓第三方软件其实就是用户自己安装的软件，区别于安装系统时自带的软件。
  • /usr/local和/opt所不同的是，/usr/local命令下面的一些子目录往往都是被加入到PATH环境变量中的，而/opt目录则没有在PATH环境变量中，这样安装在/usr/local目录下的软件就可以在命令行执行、启动。
