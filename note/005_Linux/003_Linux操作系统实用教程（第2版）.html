4.3 文件权限

  • chown：修改文件所有者

    xxxxxxxxxx
    chown ogia f1   # 将f1的所有者改为ogia
    chown ogia:ogia f1  # 同时修改所有者和所有组
    chown :ogia f1  # 只修改所有组

  • 文件权限：

    xxxxxxxxxx
    r   w   x
    4   2   1
    

    u   g   o   a

  • chmod：修改文件权限

    xxxxxxxxxx
    chmod u+x,g+w,o-r f1    # 所有者添加执行权限，组添加写权限，其他人取消读的权限
    chmod 777 f1

  • 文件默认掩码umask为0002

    • 文件创建默认权限：0666 - umask = 0664
    • 目录创建默认权限：0777 - umask = 0775

  • ln：添加文件链接

    xxxxxxxxxx
    ln -s 源文件 目标文件  # 创建软连接
    ln  # 创建硬链接

    软连接可以添加到文件或目录，而硬连接只能添加到文件，同时软连接的inode号不同，而硬链接相同。

5.1 用户和组

  • 没有指定用户的组，创建用户时，默认创建同名的组（私有组）。

  • 而标准组可以容纳多个用户。

  • 配置文件：

    • passwd：/etc/passwd，用户的账户信息。

      xxxxxxxxxx
      ogia@ogiaUbuntu:~/Desktop$ cat /etc/passwd
      root:x:0:0:root:/root:/bin/bash
      daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
      bin:x:2:2:bin:/bin:/usr/sbin/nologin
      ogia:x:1000:1000:ogia,,,:/home/ogia:/bin/bash
      # 用户名 用户密码 用户标识符（uid） 组标识符（gid） 用户注释 用户家目录 用户的bash

    • shadow：/etc/shadow，用户账号的密码信息

      xxxxxxxxxx
      ogia@ogiaUbuntu:~/Desktop$ sudo cat /etc/shadow
