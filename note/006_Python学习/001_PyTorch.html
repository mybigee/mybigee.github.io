torch.nn.Conv1d

  • 为一个类，官网。
torch.nn.Conv1d(in_channels, out_channels, kernel_size, stride=1, padding=0, dilation=1, groups=1, bias=True, padding_mode='zeros', device=None, dtype=None)
  • 使用1D卷积在由几个平面组成的输入信号。
  • 在最简单的情况下，具有（N,Ci,L）的输入和（N,Cout,Lout）的输出的输出值，可以被精确的描述为：
out(Ni,Coutj)=bias(Coutj)+∑k=0Cin−1weight(Coutj,k)∗input(Ni,k)

  • N是批量大小（batch size）、C是通道数、L是信号序列（signal sequence）的长度。

  • 该模块支持TensorFloat32。

  • 在某些ROCm设备，当使用float16输入这个模块会使用不同精度的backward。

  • stride：控制用于互相关的步幅，单个数字或单元素元组。

  • padding：控制应用于输入的填充量，该值可以是"valid"或"same"，也可以是一个整数元组，给出两边应用的隐式填充量。

  • dilation：控制内核点之间的间距，也被称为 à trous 算法。它很难描述，但这个链接有一个很好的可视化。

  • groups：控制输入和输出之间的连接。in_channels和out_channels必须可以被groups整除。

    • 当groups=1时，所有输入都卷积到所有输出。
    • 在groups=2时，该操作等效于具有两个并排的conv层，每个conv层看到一半的输入通道并产生一半的输出通道，并且随后两者被级联。
    • 在groups=in_channels，每个输入通道被卷积与自己的一组过滤（大小为out_channelsin_channels）。

  • 注意：

    • 当 groups == in_channels 和 out_channels == K * in_channels（K为正整数），该操作被称为是深度卷积。
    • 在某些情况下，当在 CUDA 设备上给定张量并使用 CuDNN 时，运算符可能会选择非确定性算法来提高性能。如果不希望这样做，可以尝试通过设置torch.backends.cudnn.deterministic = True来使操作具有确定性。
    • padding='valid'与没有填充相同。padding='same'填充输入，使输出具有作为输入的形状，但是，此模式不支持stride的值为1以外的任何值。
    • 该模型支持复杂的数据类型，如：complex32、complex64、complex128。

  • 参数：

    • in_channels（int）：输入图像中的通道数。
    • out_channels （int）：卷积产生的通道数。
    • kernel_size （int 或 tuple）：卷积内核的大小。
    • stride （int 或 tuple，可选）：卷积的步幅。默认值为1。
    • padding （int 、tuple或str，可选）：添加到输入两侧的填充。默认值为0。
    • padding_mode （str，可选）：'zeros'、 'reflect'、'replicate' 或 'circular'。默认为'zeros'。
    • dilation （int 或 tuple，可选）：内核元素之间的间距。默认为1。
    • groups （int，可选）：从输入通道到输出通道的阻塞连接数。默认为1。
    • bias （bool，可选）：如果为True，将可学习的偏差添加到输出中。默认为True。

  • 形状（shape）：

    • 输入：(N,Cin,Lin) 或 (Cin,Lin)
    • 输出：(N,Cout,Lout) 或 (Cout,Lout)
    • 其中：需要向下取整（math.floor()）
Lout=⌊Lin+2×padding−dilation×(kernel_size−1)−1stride+1⌋

  • 变量：

    • weight（Tensor）：形状（out_channels,in_channelsgroups,kernel_size）的模型的可学习权值。权值的采样从U(−k,k)，k=groupsCin∗kernel_size。
    • bias （Tensor）：形状（out_channels）的模型的可学习偏差。如果bias为True，从U(−k,k)，k=groupsCin∗kernel_size采样。

import torch
from torch import nn
import math
# 输入通道（in_channels）16，输出通道（out_channels）33，卷积核的大小（kernel_size）3
m = nn.Conv1d(16, 33, 3, stride=2)
# in_channels和C_in相等
# 表示有 20（N） 个 16（C_in） * 50（L_in）的矩阵
input = torch.randn(20, 16, 50)
output = m(input)
# (N,out_channels,L_out)
print(output.shape)
# torch.Size([20, 33, 24])
# L_out = math.floor((L_in + 2 * padding - dilation*(kernel_size - 1) - 1)/stride + 1)
# padding 默认为0，dilation默认为1，需要向下取整
print(math.floor((50 + 2*0 - 1*(3-1) - 1) / 2 + 1))
# 24

torch.nn.ConvTranspose1d(in_channels, out_channels, kernel_size, stride=1, padding=0, output_padding=0, groups=1, bias=True, dilation=1, padding_mode='zeros', device=None, dtype=None)

  • 应用1D转换卷积在由几个平面组成的输入信号。

  • 该模块可以看作是Conv1d相对于其输入的梯度。它也被称为分数步幅（fractionally-strided）卷积或反卷积（虽然它不是实际的反卷积操作，因为它不会计算卷积的真正反向）。

  • 该模块支持TensorFloat32。

  • 在某些ROCm设备，当使用float16输入这个模块会使用不同精度的backward。

  • stride：控制用于互相关的步幅。

  • padding：控制两侧的隐式零的填充量，对于dilation * (kernel_size - 1) - padding点的数量。

  • output_padding：控制添加到输出形状一侧的外加的大小。

  • dilation：控制内核点之间的间距，也被称为 à trous 算法。它很难描述，但这个链接有一个很好的可视化。

  • groups：控制输入和输出之间的连接。in_channels和out_channels必须可以被groups整除。

    • 当groups=1时，所有输入都卷积到所有输出。
    • 在groups=2时，该操作等效于具有两个并排的conv层，每个conv层看到一半的输入通道并产生一半的输出通道，并且随后两者被级联。
    • 在groups=in_channels，每个输入通道被卷积与自己的一组过滤（大小为out_channelsin_channels）。

  • 注意：

    • padding参数有效地将dilation *（kernel_size-1）- padding的填充量添加到两个大小的输入中。这被设置为使得当用相同的参数初始化Conv1d和ConvTranspose1d时，它们相对于输入和输出形状彼此相反。但是，当stride > 1时，Conv1d会将多个输入形状映射到同一个输出形状。
    • 提供output_ padding以通过在一侧有效地增加所计算的输出形状来解决这种模糊性。请注意，output_pading仅用于查找输出形状，但实际上并没有向输出添加零填充。
    • 在某些情况下，当在 CUDA 设备上给定张量并使用 CuDNN 时，运算符可能会选择非确定性算法来提高性能。如果不希望这样做，可以尝试通过设置torch.backends.cudnn.deterministic = True来使操作具有确定性。

  • 参数：

    • in_channels（int）：输入图像中的通道数。
    • out_channels （int）：卷积产生的通道数。
    • kernel_size （int 或 tuple）：卷积内核的大小。
    • stride （int 或 tuple，可选）：卷积的步幅。默认值为1。
    • padding （int 、tuple或str，可选）：dilation * (kernel_size - 1) - padding的零填充被添加到输入两侧的填充。默认值为0。
    • output_padding （str，可选）：添加额外大小到输出形状为一侧，默认为0。
    • groups （int，可选）：从输入通道到输出通道的阻塞连接数。默认为1。
    • bias （bool，可选）：如果为True，将可学习的偏差添加到输出中。默认为True。
    • dilation （int 或 tuple，可选）：内核元素之间的间距。默认为1。

  • 形状（shape）：

    • 输入：(N,Cin,Lin) 或 (Cin,Lin)
    • 输出：(N,Cout,Lout) 或 (Cout,Lout)
    • 其中：
Lout=(Lin−1)×stride−2×padding+dilation×(kernel_size−1)+output_padding+1

  • 变量：

    • weight（Tensor）：形状（in_channels,out_channelsgroups,kernel_size）的模型的可学习权值。权值的采样从U(−k,k)，k=groupsCout∗kernel_size。
    • bias （Tensor）：形状（out_channels）的模型的可学习偏差。如果bias为True，从U(−k,k)，k=groupsCout∗kernel_size采样。

  • 在最简单的情况下，具有（N,Cin,H,W）的输入和（N,Cout,Hout,Wout）的输出的输出值，可以被精确的描述为：

  • 该模块支持TensorFloat32。

  • 在某些ROCm设备，当使用float16输入这个模块会使用不同精度的backward。

  • stride：控制用于互相关的步幅，单个数字或单元素元组。

  • padding：控制应用于输入的填充量，该值可以是"valid"或"same"，也可以是一个整数元组，给出两边应用的隐式填充量。

  • dilation：控制内核点之间的间距，也被称为 à trous 算法。它很难描述，但这个链接有一个很好的可视化。

    • 单一的int，此时高度和宽度维度使用相同的值。

    • 该模型支持复杂的数据类型，如：complex32、complex64、complex128。

  • 参数：

    • in_channels（int）：输入图像中的通道数。
    • out_channels （int）：卷积产生的通道数。
    • kernel_size （int 或 tuple）：卷积内核的大小。
    • stride （int 或 tuple，可选）：卷积的步幅。默认值为1。
    • padding （int 、tuple或str，可选）：添加到输入两侧的填充。默认值为0。
    • padding_mode （str，可选）：'zeros'、 'reflect'、'replicate' 或 'circular'。默认为'zeros'。
    • dilation （int 或 tuple，可选）：内核元素之间的间距。默认为1。
    • groups （int，可选）：从输入通道到输出通道的阻塞连接数。默认为1。
    • bias （bool，可选）：如果为True，将可学习的偏差添加到输出中。默认为True。

  • 形状（shape）：

    • 输入：(N,Cin,Hin,Win) 或 (Cin,Hin,Win)
    • 输出：(N,Cout,Hout,Wout) 或 (Cout,Hout,Wout)
