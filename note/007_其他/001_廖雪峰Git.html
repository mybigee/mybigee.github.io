廖雪峰Git常用命令注意

常用

  • 初始化：git init
  • 添加文件：git add 文件名
  • 查看状态：git status
  • 提交：git commit -m "描述文字"
  • 添加用户邮箱：git config --global user.email "xxxxxx@qq.com"
  • 添加用户：git config --global user.name "mybigee"
  • 查看文件版本号：git log
  • 仅显示版本号与描述性文字：git log --pretty=oneline
  • 回到之前版本：git reset --hard + 版本号
  • 克隆远程仓库：git clone git@github.com:用户名/git库名.git
  • 复制：shift + insert

命令

  1. 把当前目录变成可管理仓库：git init

  2. 添加文件到仓库：git add 文件名

  3. 提交文件：git commit -m "说明文字（可以为任意值）"

  4. 显示仓库的状态：git status

  5. 查看具体怎么修改文件的：git diff 文件名(diff即为different)

  6. 查看每个提交的版本：git log

    仅显示版本号与描述性文字：git log --pretty=oneline

  7. 回到某一个版本：git reset --hard 版本号

  8. 查看用过的每一条记录：git reflog

  9. 查看工作区和版本库里面最新版本的区别：git diff HEAD -- 文件名

  10. 撤销修改：git checkout -- 文件名(下面13条)

  11. 撤销暂存区（只add 未commit）：git reset HEAD 文件名

  12. 关联远程库：git remote add origin(远程库名) git@github.com:用户名/git库名.git（下面15条）

    第一次将本地库的所有内容推到远程库：git push -u origin master

    非第一次提交到远程库：git push origin master

    删除远程库：git remote rm origin（下面16条）

    克隆远程仓库：git clone git@github.com:用户名/git库名.git

  13. 创建并且换到分支：git checkout -b dev(分支名)

    创建分支：git branch dev

    切换分支：git checkout dev

    查看所有/当前分支：git branch (下面17条)

    合并dev分支到当前分支：git merge dev(被合并的分支名)

    删除分支：git branch -d dev

    创建并且换到分支(更好)：git switch -c dev

    切换到master：git switch master

  14. 查看分支合并图：git log --graph [--pretty=oneline --abbrev-commit]

  15. 取消fast forward模式合并：git merge --no-ff -m "描述文字" 分支名(下面19条)

  16. 隐藏工作区：git stash

    恢复工作区：git stash apply

    恢复后删除stash内容：git stash drop

    恢复并同时删除stash内容：git stash pop

    查看stash：git stash list

    复制特定提交到当前分支：git cherry-pick 版本号

  17. 强行删除没有合并过的分支：git branch -D 分支名

  18. 打标签：git tag v1.0(标签名)(下面20条)

    查看标签：git tag

    为非最新的commit打标签：git tag 标签名 版本号 (下面21条)

    查看标签信息：git show 标签名 (下面22条)

    创建带有说明的标签：git tag -a 标签名 -m "描述文字" 版本号

    删除标签：git tag -d 标签名

    推送到远程：git push origin(远程仓库名) 标签名

    一次性推送所有未推送的标签：git push origin --tags

    已推送到远程的标签的删除：git push origin :refs/tags/标签名(要先删除本地的标签)

  19. 强制添加文件到git：git add -f 文件名

  20. 配置别名：git config --global alias.别名 原标签名 (下面24条)

  21. 查看每个仓库的配置文件：cat .git/config

  22. 当前用户的配置文件：cat .gitconfig

注意

  1. Microsoft的Word格式是二进制格式，版本控制系统是没法跟踪Word文件的改动的，要以纯文本方式编写文件。

  2. git commit 执行后会显示：n file changed(n个文件被改动)，m insertions(插入了m行内容)。

  3. git add 可以一次提交多个文件，用空格隔开。

  4. 输入git status 后会显示未添加的文件(在Untracked files 中)。

  5. 每次提交修改文件都要git add，要不断的修改提交。

  6. 使用git log 时只有用git add 与git commit 后才可以查到。

  7. HEAD 表示当前版本，HEAD^表示上一版本，HEAD^^表示上上一版本，也可以写成HEAD~n

  8. 回到以前版本后如果当前窗口已关掉，可以输入git reflog 来找到输入的每条命令。如果未关掉，往回找找到版本号即可。

  9. 版本号可以不写全，只写前面几位即可。

  10. 添加文件的步骤：先将工作区（操作的文件夹）的文件用 git add 添加到暂存区(stage)，然后再一次性通过 git commit 提交到当前分支（创建时只有一个未master），而HEAD是一个指向master的指针。

  11. Git跟踪并管理的是修改，而非文件。

  12. 修改后必须add 才可以commit，否则add之后的不会别提交。

  13. git checkout：分为两种情况：

    • 一是文件自修改后还没有被放到暂存区，撤销修改就回到和版本库一模一样的状态；
    • 二是文件已经添加到暂存区后，又作了修改，撤销修改就回到添加到暂存区后的状态。

  14. 删除文件（rm）后，可以用 status 查看被删除的文件，此时工作区和版本库就不一致了，需要用commit使他们同步。如果要恢复得用checkout。

  15. 关联后，origin即为远程库名。

  16. 删除前先用 git remote -v 查看远程库信息，然后再删除。

  17. 使用git branch后会列出所有分支，当前分支前面有*号。

  18. 文件合并冲突。

  19. fast forward模式(一般为默认模式)会在合并并删除分支后，丢弃分支信息(将分支信息完全删除)，--no-ff 可以禁用该模式，同时还要生成新的commit，所以要加上-m。

  20. 遇到无法切回master时，试一试先add/commit 后再切换。

  21. 打标签时要先切换回要打标签的地方。默认标签是打在最新提交的commit上的，要打在之前的commit上需要找到版本号。

  22. 标签不是按时间顺序列出，而是按字母排序的。

  23. 标签总是和某个commit挂钩。如果这个commit既出现在master分支，又出现在dev分支，那么在这两个分支上都可以看到这个标签。

  24. --global参数是全局参数，也就是这些命令在这台电脑的所有Git仓库下都有用。如果不加，那只针对当前的仓库起作用。
